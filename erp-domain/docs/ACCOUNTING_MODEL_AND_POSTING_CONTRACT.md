# Accounting Model + Posting Contract

## Model Inventory (entities and linkage)
- accounts: company, code, type, balance.
- journal_entries: company, reference_number, entry_date, status, currency, fx_rate, accounting_period_id,
  dealer_id, supplier_id, reversal_of_id, audit columns.
- journal_lines: journal_entry_id, account_id, debit, credit, description.
- accounting_periods: company, year/month, status, closing_journal_entry_id.
- dealer_ledger_entries, supplier_ledger_entries: journal_entry_id, debit, credit, reference_number.
- partner_settlement_allocations: journal_entry_id plus invoice/purchase links.
- accounting_events: event store with journal_entry_id, account_id, journal_reference.
- journal_reference_mappings: legacy <-> canonical journal references per company.
- source document links to journal entries:
  - invoices.journal_entry_id
  - raw_material_purchases.journal_entry_id
  - payroll_runs.journal_entry_id and payroll_runs.journal_entry_ref_id
  - packaging_slips.journal_entry_id and packaging_slips.cogs_journal_entry_id
  - inventory_movements.journal_entry_id and raw_material_movements.journal_entry_id
  - inventory_adjustments.journal_entry_id

## Posting Contract (canonical)
### Required context
- Company context is required (X-Company-Id + authenticated user).
- entryDate is required and validated against CompanyClock.
- referenceNumber is required or generated by ReferenceNumberService and unique per company.
- currency defaults to company base currency; fxRate is required for non-base currency.

### Balancing rules
- Each line must have exactly one of debit/credit and non-negative amounts.
- Total debits must equal total credits in base currency.
- Minor FX rounding deltas within tolerance may be absorbed into a single line.

### Partner context
- If a dealer or supplier control account is posted, dealerId/supplierId must be present and match.
- Dealer and supplier ledger entries are recorded from the net AR/AP control lines.

### Period enforcement
- When period lock is enforced, entryDate must map to an OPEN period.
- Closed or locked periods reject new postings; reversals honor the same rules.
- Admin override is required to bypass date window validation and closed period reversals.

### Idempotency and reference mapping
- Journal entry references are unique per company.
- Duplicate references are idempotent only when entryDate, partner, currency/fxRate, and line signatures match.
- JournalReferenceMapping resolves legacy references (for example SALE- to INV-).
- Upstream flows must pass stable references (idempotencyKey or deterministic referenceNumber).

### Opening balances (opening stock)
- Opening stock import posts a single OPEN-STOCK journal entry.
- Debit inventory control accounts (per item or company default) and credit OPEN-BAL equity.
- Opening stock movements must carry the journal_entry_id for audit linkage.

### Linkage expectations
- Posting services must persist the created journal entry ID on the source document.
- Inventory movements created from a posting must carry journal_entry_id.
- Reversals must set reversal_of_id and reversal_entry_id for traceability.

### Audit trail
- created_by, posted_by, posted_at, and correction_reason are required for traceability.
- AccountingEventStore captures posted and reversed events for audit replay.

## Performance Notes (accounting hot paths)
- Journal queries rely on composite indexes by company/date/status and partner filtering.
- Ledger statements/aging rely on supplier/dealer + entry_date indexes.
- Journal lines benefit from journal_entry_id and account_id indexes.
